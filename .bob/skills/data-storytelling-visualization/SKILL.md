# Data Storytelling & Visualization

## Overview

This skill provides 5 systematic approaches to turn raw findings into insights that drive decisions:

1. **Insight Synthesis** - Structure analysis outputs into clear business insights
2. **Visualization Builder** - Chart type selection, design guidance, and spec generation
3. **Executive Summary Generator** - Concise executive-ready summaries of complex analysis
4. **Dashboard Specification** - Full dashboard requirements with metrics and layout
5. **Data Narrative Builder** - Craft a compelling story arc from analytical findings

**Core Philosophy**: Data doesn't speak for itself. Transform analysis into stories that drive action.

---

## When to Use This Skill

Use this skill when:
- **Synthesizing findings**: Turn raw analysis into actionable insights
- **Creating visualizations**: Choose the right chart type and design
- **Presenting to executives**: Create concise, high-impact summaries
- **Building dashboards**: Specify metrics, layout, and interactions
- **Telling data stories**: Craft compelling narratives from data
- **Driving decisions**: Make findings memorable and actionable

## Activity 1: Insight Synthesis

**Purpose**: Structure analysis outputs into clear business insights.

### When to use
- You've completed an analysis and have many findings but no clear "so what?"
- You need to distill complex analysis into 3-5 key takeaways
- Stakeholders are asking "what should we do with this information?"
- You want to ensure your analysis leads to action

### Process
1. **List all findings** - Document every observation from your analysis.
2. **Identify patterns** - Group related findings into themes.
3. **Assess business impact** - Evaluate which findings matter most to business goals.
4. **Craft insight statements** - Write clear, actionable insights (observation + implication + recommendation).
5. **Prioritize insights** - Rank by impact and urgency.
6. **Add supporting evidence** - Link each insight to specific data points.
7. **Create action items** - Convert insights into concrete next steps.

### Inputs needed
- **Required**: Analysis results and findings
- **Required**: Business context and goals
- **Optional**: Stakeholder priorities
- **Optional**: Constraints or limitations

### Output
- 3-5 prioritized insights
- Supporting evidence for each insight
- Recommended actions
- Impact assessment

## Activity 2: Visualization Builder

**Purpose**: Chart type selection, design guidance, and specification generation.

### When to use
- You need to choose the right chart type for your data
- You want to ensure visualizations follow best practices
- You're creating specifications for a designer or developer
- You need to critique or improve existing visualizations

### Process
1. **Understand the message** - What story are you trying to tell with this visualization?
2. **Assess the data** - What type of data do you have (categorical, continuous, time series, etc.)?
3. **Choose chart type** - Select the most appropriate visualization (bar, line, scatter, heatmap, etc.).
4. **Apply design principles** - Use color, labels, and layout effectively.
5. **Add context** - Include titles, annotations, and reference lines.
6. **Test readability** - Ensure the visualization is clear and not misleading.
7. **Create specification** - Document chart type, data mappings, styling, and interactions.

### Inputs needed
- **Required**: Data to visualize
- **Required**: Message or insight to communicate
- **Optional**: Brand guidelines or style requirements
- **Optional**: Target audience

### Output
- Chart type recommendation
- Design specification
- Data mappings
- Styling and interaction details

## Activity 3: Executive Summary Generator

**Purpose**: Concise executive-ready summaries of complex analysis.

### When to use
- You need to present analysis to senior leadership
- You have 5 minutes (or less) to communicate findings
- You want to create a one-page summary of a complex analysis
- You need to get executive buy-in for recommendations

### Process
1. **Identify the key message** - What's the single most important thing executives should know?
2. **Structure the summary** - Use: Situation → Complication → Resolution → Action.
3. **Write the headline** - Create a clear, compelling title that captures the main finding.
4. **Summarize findings** - Distill analysis into 3-5 bullet points.
5. **Quantify impact** - Use numbers to show business impact (revenue, cost, time, etc.).
6. **Recommend actions** - Provide clear, specific next steps.
7. **Add visual anchor** - Include one key chart or metric that tells the story.

### Inputs needed
- **Required**: Analysis findings and insights
- **Required**: Business context and goals
- **Optional**: Executive priorities or concerns
- **Optional**: Time or budget constraints

### Output
- One-page executive summary
- Key headline and findings
- Impact quantification
- Recommended actions
- Supporting visualization

## Activity 4: Dashboard Specification

**Purpose**: Full dashboard requirements with metrics, layout, and interactions.

### When to use
- You're designing a new dashboard from scratch
- You need to specify requirements for a developer or BI tool
- You want to ensure a dashboard meets user needs
- You're redesigning an existing dashboard

### Process
1. **Define the audience** - Who will use this dashboard and what decisions do they need to make?
2. **Identify key metrics** - What metrics are most important for this audience?
3. **Design the layout** - Organize metrics into logical sections with visual hierarchy.
4. **Specify interactions** - Define filters, drill-downs, and dynamic elements.
5. **Choose visualizations** - Select appropriate chart types for each metric.
6. **Add context** - Include benchmarks, targets, and trend indicators.
7. **Document requirements** - Create a complete specification for implementation.

### Inputs needed
- **Required**: Dashboard purpose and audience
- **Required**: Available data and metrics
- **Optional**: Existing dashboards to reference or replace
- **Optional**: Technical constraints (BI tool, data refresh frequency)

### Output
- Dashboard specification document
- Metric definitions and calculations
- Layout and visual hierarchy
- Interaction specifications
- Implementation notes

## Activity 5: Data Narrative Builder

**Purpose**: Craft a compelling story arc from analytical findings.

### When to use
- You need to present analysis in a meeting or presentation
- You want to make findings memorable and persuasive
- You're writing a report or blog post about your analysis
- You need to build a case for a recommendation

### Process
1. **Set the scene** - Establish context: What's the business situation or problem?
2. **Introduce tension** - Present the complication: What's not working or what changed?
3. **Build the investigation** - Show how you analyzed the problem (your analytical journey).
4. **Reveal insights** - Present key findings with supporting evidence.
5. **Show implications** - Explain what the findings mean for the business.
6. **Propose resolution** - Recommend specific actions to address the situation.
7. **Create the narrative flow** - Arrange elements into a logical, compelling story.

### Inputs needed
- **Required**: Analysis findings and insights
- **Required**: Business context and stakeholder concerns
- **Optional**: Historical context or previous attempts to solve the problem
- **Optional**: Success stories or case studies

### Output
- Complete data narrative
- Story arc with clear beginning, middle, and end
- Supporting visualizations
- Call to action

## Key Principles

1. **Start with the message** - Know what you want to communicate before choosing visualizations
2. **Less is more** - Remove unnecessary elements; focus on what matters
3. **Show, don't tell** - Use visualizations to make insights obvious
4. **Make it actionable** - Every insight should lead to a decision or action
5. **Know your audience** - Tailor complexity and detail to audience needs
6. **Tell a story** - Create a narrative arc that engages and persuades

## Visualization Best Practices

### Chart Type Selection
- **Comparison**: Bar charts, grouped bars
- **Trend over time**: Line charts, area charts
- **Distribution**: Histograms, box plots
- **Relationship**: Scatter plots, bubble charts
- **Composition**: Stacked bars, pie charts (use sparingly)
- **Geographic**: Maps, choropleth maps

### Design Principles
- Use color purposefully (highlight key data, show categories)
- Label directly on charts when possible
- Start y-axis at zero for bar charts
- Use consistent scales for comparison
- Add reference lines for context (targets, benchmarks)
- Remove chart junk (unnecessary gridlines, 3D effects, etc.)

### Common Mistakes to Avoid
- Pie charts with too many slices
- Dual y-axes that mislead
- 3D charts that distort perception
- Truncated y-axes that exaggerate differences
- Too many colors or patterns
- Missing context (no title, labels, or units)
